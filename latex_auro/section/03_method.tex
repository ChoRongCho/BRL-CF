\section{Method}
\label{sec:method}

\subsection{Problem Formulation}

The robot is given a symbolic domain, a set of executable skills, an initial knowledge state, and a task goal. Some facts are known with certainty, while local task facts must be inferred from observations. The robot selects task actions $a \in \mathcal{A}$ using POMCP. After each action, the observation may support several possible successor states rather than a single deterministic outcome. The system then computes belief uncertainty and requests human feedback only if the updated belief is not confident enough to commit a symbolic state to the knowledge base. The objective is to maximize task reward while reducing unnecessary feedback requests:
\begin{equation}
\max_{\pi} \mathbb{E}_{\pi}\left[\sum_{t=0}^{T}\gamma^t
\left(R(s_t,a_t)-\lambda \mathbb{I}[\mathrm{query}_t]\right)\right],
\label{eq:objective}
\end{equation}
where $\mathrm{query}_t$ indicates that feedback is requested after the belief update at time $t$, and $\lambda$ is the cost of human interaction.

\subsection{Knowledge Base}

The knowledge base is partitioned into domain information, static information, and local information:
\begin{equation}
\mathcal{KB}=\mathcal{I}^{D}\sqcup \mathcal{I}^{S}\sqcup \mathcal{I}^{L}.
\end{equation}
Domain information encodes predicates, action schemas, task rules, and robot skills. Static information contains facts and numeric fluents that remain fixed during an episode, such as object identifiers, bin locations, or stem locations. Local information contains facts inferred during execution, such as object category, tomato state, object pose, and visibility. Uncertainty is concentrated in $\mathcal{I}^{L}$ because these facts depend on perception and partial observations.

\subsection{Symbolic Frontier Belief}

Let $s^K$ denote the current committed knowledge state. Instead of constructing a global belief over all symbolic states, we construct a local belief over states reachable from $s^K$ under the selected task action. For action $a$, the symbolic frontier is
\begin{equation}
\mathrm{Frontier}(s^K,a)=\{s_1^\star,\ldots,s_j^\star\},
\label{eq:frontier}
\end{equation}
where each $s_k^\star$ is a reachable successor hypothesis consistent with the action model and the current knowledge base. The belief over the frontier is represented by weighted particles:
\begin{equation}
\mathcal{B}=\{(s_k^\star,w_k)\}_{k=1}^{j}, \qquad \sum_{k=1}^{j}w_k=1.
\label{eq:belief_particles}
\end{equation}
After executing $a$ and receiving observation $o$, each particle weight is updated by the observation model:
\begin{equation}
w'_k = w_k O(o|s_k^\star,a), \qquad
\tilde{w}_k = \frac{w'_k}{\sum_{\ell=1}^{j}w'_\ell}.
\label{eq:frontier_update}
\end{equation}

This local frontier construction has two roles. First, it avoids enumerating irrelevant symbolic worlds. Second, it ties uncertainty to the action currently affecting task progress, which makes later human queries task-relevant.

\subsection{Confidence and Query Trigger}

We quantify uncertainty using the entropy of the normalized frontier belief:
\begin{equation}
H(\mathcal{B})=-\sum_{k=1}^{j}\tilde{w}_k\log_2 \tilde{w}_k.
\label{eq:entropy}
\end{equation}
The confidence score is
\begin{equation}
\mathrm{Conf}(\mathcal{B})=
\begin{cases}
1, & j=1,\\
1-\frac{H(\mathcal{B})}{\log_2 j}, & j>1.
\end{cases}
\label{eq:confidence}
\end{equation}
If $\mathrm{Conf}(\mathcal{B})\geq \tau$, the robot commits the maximum-weight state to the knowledge base:
\begin{equation}
s^K \leftarrow \arg\max_{s_k^\star \in \mathcal{B}}\tilde{w}_k.
\label{eq:commit}
\end{equation}
If $\mathrm{Conf}(\mathcal{B})<\tau$, the robot requests human feedback. The threshold $\tau$ is therefore an autonomy-interaction parameter: lower values allow the robot to act with less certainty, while higher values increase interaction to obtain more reliable knowledge.

\subsection{Query Selection}

The system constructs candidate hypotheses from literals whose truth values differ across the frontier states. Let $\mathcal{C}$ be this set. For each candidate $c\in \mathcal{C}$, the belief is partitioned into states where $c$ is true and false:
\[
\mathcal{B}^{+}_{c}=\{(s_k^\star,\tilde{w}_k)\mid c\in s_k^\star\}, \quad
\mathcal{B}^{-}_{c}=\{(s_k^\star,\tilde{w}_k)\mid c\notin s_k^\star\}.
\]
The selected query minimizes expected posterior entropy:
\begin{equation}
c^*=\arg\min_{c\in \mathcal{C}}
\left[
P(c)H(\mathcal{B}^{+}_{c})+
P(\neg c)H(\mathcal{B}^{-}_{c})
\right],
\label{eq:query_selection}
\end{equation}
where $P(c)=\sum_{s_k^\star:c\in s_k^\star}\tilde{w}_k$ and $P(\neg c)=1-P(c)$. After the human answers, the belief is filtered:
\begin{equation}
\mathcal{B}\leftarrow
\begin{cases}
\mathcal{B}^{+}_{c^*}, & y=\texttt{true},\\
\mathcal{B}^{-}_{c^*}, & y=\texttt{false}.
\end{cases}
\label{eq:filter}
\end{equation}
The remaining weights are renormalized. Querying repeats until the confidence exceeds $\tau$ or no informative hypothesis remains.

\subsection{Planning and Execution Pipeline}

\placeholderfig{Detailed system architecture connecting the knowledge base, sensing and perception module, POMCP planning manager, feedback manager, user interface, and robot execution layer.}{2.0in}{fig:architecture}

At each step, the planning manager runs POMCP using the current belief and the symbolic action constraints. The selected task action is executed by a corresponding robot skill. Perception produces observations used to update the frontier belief. If the updated confidence is low, the feedback manager converts the selected symbolic hypothesis into a human-readable query, highlights the relevant object in the interface, receives a binary response, and returns the response to the planner. The knowledge base is updated only after the belief is sufficiently confident.

\begin{algorithm}[t]
\caption{Uncertainty-Aware Planning with Human Querying}
\label{alg:planning}
\KwIn{initial knowledge state $s^K_0$, threshold $\tau$}
\KwOut{executed action sequence}
$s^K \leftarrow s^K_0$\;
\For{$t=0$ \KwTo $T$}{
    $a_t \leftarrow \mathrm{POMCP}(s^K,\mathcal{B})$\;
    \If{$a_t=\emptyset$}{\textbf{break}}
    execute $a_t$ and receive observation $o_{t+1}$\;
    construct $\mathrm{Frontier}(s^K,a_t)$ and update $\mathcal{B}$ using Eq.~\ref{eq:frontier_update}\;
    compute $\mathrm{Conf}(\mathcal{B})$ using Eq.~\ref{eq:confidence}\;
    \While{$\mathrm{Conf}(\mathcal{B}) < \tau$}{
        select $c^*$ using Eq.~\ref{eq:query_selection}\;
        query human for $c^*$ and receive $y$\;
        filter $\mathcal{B}$ using Eq.~\ref{eq:filter}\;
        recompute $\mathrm{Conf}(\mathcal{B})$\;
    }
    commit the most likely state to $s^K$ using Eq.~\ref{eq:commit}\;
}
\end{algorithm}
