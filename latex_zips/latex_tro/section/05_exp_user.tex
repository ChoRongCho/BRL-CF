\subsection{User Study}~\label{exp:user_study}
We conducted a Wizard-of-Oz (WoZ) user study as a preliminary evaluation of how different query styles affect human task performance and interaction burden. The study complements the system evaluation by testing whether state-level queries remain usable for human operators in the waste sorting and tomato harvesting domains. Because the study includes $12$ participants and each participant experienced each domain--condition pair once, we treat the results as user-facing evidence for interaction trends rather than as a standalone proof of superiority. We focus on four measures: task success, operation time, NASA Raw-TLX workload, and SAGAT accuracy.

\subsubsection{Participants}~\label{user:participants}
Twelve participants (7 female and 5 male) took part in the study. Participants ranged in age from 21 to 26 years ($M=23.0$, $SD=1.48$). Each participant received \$20 upon completion of the study.

\subsubsection{Study Design and Conditions}~\label{user:design}
The study employed a within-subject design. Each participant completed ten trials in total, consisting of five waste sorting scenarios and five tomato harvesting scenarios. In each domain, participants experienced five interaction conditions: \textbf{All}, \textbf{No}, \textbf{Ours1}, \textbf{Ours2}, and \textbf{KnowNo}. \textbf{All} requested feedback whenever a query was available, representing continuous human supervision. \textbf{No} performed the task without requesting user feedback. \textbf{Ours1} used the proposed uncertainty-based state query policy with the same interaction logic as the system-level \textbf{Ours} policy. \textbf{Ours2} used the same state-level query interface with a different WoZ scenario profile to expose participants to a broader range of state-query situations. \textbf{KnowNo} served as an action-level query baseline, in which users were asked to select the robot's next action rather than verify a task-level state hypothesis.

Each domain consisted of five scenarios and five interaction conditions. The assignment between scenarios and conditions was randomized for each participant such that every participant experienced all five conditions exactly once per domain. Detailed scenario descriptions are provided in Appendix~\ref{appx:user}.

\subsubsection{Materials and Procedure}~\label{user:materials_procedure}
Participants interacted with the robot through a tablet-based interface. The interface displayed the current task state, visual sensing information, task-relevant objects, and robot-generated questions with selectable answer options. The waste sorting scenarios required participants to identify waste categories, including general waste, paper, cans, and plastic. The tomato harvesting scenarios required participants to identify tomato states and support harvesting decisions, such as whether a target tomato was present, ripe, unripe, or rotten.

Before the experiment, participants received an explanation of the two task domains and completed practice interactions with the tablet interface. During the main session, they completed ten trials without assistance. If a participant selected an incorrect answer, the response was recorded as incorrect for evaluation purposes, but the scenario continued using the intended task state so that later interactions were not affected by earlier mistakes. After each scenario, participants completed questionnaire items for workload and situation awareness.

\subsubsection{Measures and Analysis}~\label{user:measures}
Task success measured whether participants provided all required responses correctly within a task. \textbf{No} is omitted from task success because it contains no scorable user responses. Operation time measured active user operation time after subtracting passive scenario video duration; \textbf{No} is also omitted from this measure because it involves no active interaction. Workload was measured using NASA Raw-TLX. We still report \textbf{No} for workload because no feedback is expected to impose the least interaction burden, but this condition does not provide user corrections to the robot. Situation awareness was measured using SAGAT-style questions, grouped into Lv1 perception, Lv2 comprehension, and Lv3 projection.

For condition-level comparisons, we used Cochran's $Q$ test for task success and Friedman tests for operation time and workload. Given the small sample size and one trial per condition in each domain, we use these tests to qualify the descriptive results rather than to make broad claims about population-level effects. The statistical tests are exploratory and are reported without correction for multiple comparisons; pairwise post-hoc tests are not used to claim condition-level superiority.

\subsubsection{Task Success and Operation Time}~\label{user:results_task_time}
Table~\ref{tab:user_task_time} and Fig.~\ref{fig:user_task_time_results} summarize task success and operation time. In waste sorting, task success differed across conditions, $Q(3)=27.69$, $p<.001$. Descriptively, \textbf{Ours1} was comparable to \textbf{All} in success ($91.7\%$ for both), and \textbf{Ours2} reached $100.0\%$. These success rates were obtained with substantially shorter operation time: \textbf{All} required $30.0$~s, whereas \textbf{Ours1} and \textbf{Ours2} required $11.2$~s and $10.0$~s, respectively. Operation time also differed across conditions in waste sorting, $\chi^2(3)=15.61$, $p=.002$. \textbf{KnowNo} achieved only $8.3\%$ success and required $29.2$~s, suggesting that action-level queries were difficult for users in this waste-sorting setting.

The tomato harvesting domain showed a different trade-off. The condition effect on task success was not significant, $Q(3)=5.40$, $p=.143$. \textbf{Ours1} and \textbf{Ours2} both achieved $75.0\%$ success, compared with $66.7\%$ for \textbf{All}. \textbf{KnowNo} achieved the highest success rate in this domain ($100.0\%$), indicating that action-level guidance can be effective when the next action is visually and temporally constrained. However, \textbf{KnowNo} required $25.9$~s of operation time, whereas \textbf{Ours1} required $21.4$~s and \textbf{Ours2} required $17.6$~s. The condition effect on tomato operation time was not significant, $\chi^2(3)=6.57$, $p=.085$, so we interpret this result as an efficiency trend rather than a decisive advantage.

\subsubsection{Workload and SAGAT}~\label{user:results_workload_sa}
Fig.~\ref{fig:user_workload_results} and Table~\ref{tab:user_workload_sa} summarize NASA Raw-TLX workload. Workload differed across conditions in both waste sorting, $\chi^2(4)=21.31$, $p<.001$, and tomato harvesting, $\chi^2(4)=16.31$, $p=.003$. As expected, \textbf{No} produced the lowest workload in both domains ($1.65$ in waste sorting and $1.72$ in tomato harvesting), because it did not ask users to provide corrective feedback. This low workload should therefore be interpreted together with the absence of task support.

Among feedback-based conditions, \textbf{Ours2} produced the lowest workload in waste sorting ($1.89$), lower than \textbf{All} ($2.06$), \textbf{Ours1} ($2.26$), and \textbf{KnowNo} ($2.54$). In tomato harvesting, \textbf{Ours1} and \textbf{Ours2} reduced workload relative to \textbf{All}: $1.89$ and $1.81$ versus $2.21$. \textbf{KnowNo} also scored $1.81$ in tomato harvesting, consistent with its strong success rate in that domain. These results suggest that state-level queries can reduce perceived workload compared with continuous feedback, while action-level queries may also be efficient when the action choice is straightforward.

\subsubsection{SAGAT Results}~\label{user:results_sagat_levels}
Table~\ref{tab:user_sagat_levels} reports SAGAT accuracy by level. The SAGAT results are mixed and should not be read as uniformly favoring one condition. In waste sorting, \textbf{Ours1} produced the highest Lv2 comprehension score ($0.63$) and a high Lv3 projection score ($0.75$), compared with \textbf{All} ($0.38$ for Lv2 and $0.38$ for Lv3) and \textbf{KnowNo} ($0.46$ for Lv2 and $0.58$ for Lv3). However, \textbf{Ours2} had a low Lv2 score ($0.08$), despite maintaining high Lv1 perception ($0.88$) and moderate Lv3 projection ($0.63$). This indicates that the Ours2 query profile did not consistently support comprehension-level awareness in the waste sorting task.

In tomato harvesting, \textbf{Ours2} had the highest descriptive Lv3 projection score ($0.96$), followed by \textbf{Ours1} and \textbf{No} ($0.83$), \textbf{All} ($0.71$), and \textbf{KnowNo} ($0.63$). The relatively high Lv3 scores for \textbf{No} in both domains should be interpreted cautiously: participants could observe the scenario without active intervention, but this condition did not provide corrective support to the robot and is not included in task-success scoring. At the same time, \textbf{KnowNo} achieved the highest task success in tomato harvesting, so the user-study evidence is better summarized as a trade-off: state-level queries reduced operation time and workload, while action-level queries were highly effective in the tomato task. Overall, the SAGAT results suggest that state-level queries can preserve or improve some aspects of situation awareness, but the effect depends on the domain and the level of awareness being measured.

\subsubsection{Summary}~\label{user:discussion}
The user study supports a more specific claim than overall dominance: state-level queries can reduce interaction effort while maintaining task performance descriptively comparable to continuous feedback. In waste sorting, \textbf{Ours1} and \textbf{All} both achieved $91.7\%$ success, and \textbf{Ours2} reached $100.0\%$, while operation time decreased from $30.0$~s under \textbf{All} to $11.2$~s and $10.0$~s under the two proposed conditions. In tomato harvesting, \textbf{KnowNo} achieved the highest success rate ($100.0\%$), showing that action-level queries can be effective in visually constrained action-selection settings. The proposed state-query conditions instead provided a lower-interaction alternative, reducing operation time relative to \textbf{All} and \textbf{KnowNo} while maintaining $75.0\%$ success.

The workload and SAGAT results reinforce this trade-off. \textbf{No} naturally produced the lowest workload because it did not request feedback, but it also provided no corrective support. Among feedback-based policies, the proposed conditions reduced workload relative to continuous feedback in tomato harvesting and achieved the lowest feedback-based workload in waste sorting through \textbf{Ours2}. SAGAT accuracy did not uniformly favor the proposed method: \textbf{Ours1} improved waste-domain Lv2/Lv3 scores, whereas \textbf{Ours2} showed a clear weakness in waste-domain Lv2. The relatively high Lv3 scores under \textbf{No} further suggest that passive observation can support some projection questions, even when the condition provides no active task assistance. These mixed results indicate that query design affects not only effort but also the type of situation awareness supported. Taken together, the user study provides preliminary evidence that state-level queries are a practical way to reduce user effort, while also identifying domain-dependent trade-offs that should be examined in larger user studies.
