
\section{Introduction}~\label{intro}

Recent advances in robotic systems have significantly improved autonomy in perception, manipulation, and planning across domains such as agriculture and household environments. Despite these advances, robotic systems operating in real-world environments still encounter perception errors~\cite{kaipa2016addressing}, action execution failures~\cite{kaelbling1998planning}, and unexpected environmental changes~\cite{kurniawati2016online}. As a result, many systems continue to rely on human operators to monitor execution and intervene when problems occur~\cite{dragan2013legibility, goodrich2008human}.

While human supervision can improve reliability, continuously monitoring robot execution imposes substantial cognitive demands on users, particularly during long-horizon tasks~\cite{chen2014human}. In practice, users are often asked to oversee robot behavior even when intervention is unnecessary, resulting in increased workload, reduced attention, and inefficient human--robot collaboration. Rather than requiring users to continuously assess the correctness of robot decisions, robots should be able to identify situations in which autonomous reasoning is insufficient and request assistance only when human input is likely to be beneficial~\cite{sadigh2017active, leusmann2023understanding}. This raises two fundamental questions for collaborative robotic systems: (i) \textit{when} should a robot request assistance, and (ii) \textit{what} information should it request from the user?

To address these questions, we present a knowledge-based robotic framework that enables robots to selectively seek human assistance under uncertainty. We formulate robot decision making as a POMDP~\cite{kaelbling1998planning} and represent knowledge using a STRIPS-style symbolic model~\cite{fikes1971strips}. The system maintains a belief over alternative world hypotheses and continuously updates this belief using observations collected during execution. By estimating uncertainty from the belief distribution, the robot determines whether additional human input is required. When assistance is needed, the system generates queries that are expected to best distinguish among competing hypotheses. Consequently, users are only asked to provide simple environmental feedback when the robot cannot confidently resolve uncertainty on its own, eliminating the need for continuous supervision or direct involvement in planning decisions.

Fig.~\ref{fig:intro} illustrates the overall system architecture. The proposed framework integrates sensing, planning, feedback management, and user interaction through a centralized knowledge base. The robot maintains and updates its belief using both sensor observations and user feedback, allowing it to reason over uncertain world states while preserving an interpretable symbolic representation. This architecture enables information to be shared consistently across modules and supports selective human assistance during execution.

We evaluate the proposed system in two representative robotic domains, tomato harvesting and waste sorting, through simulation, real-robot experiments, and user studies. Results show that the proposed approach substantially reduces the amount of required human intervention while maintaining task performance. Furthermore, user studies demonstrate reduced cognitive workload compared to conventional supervision-intensive approaches.

The main contributions of this work are as follows:

\begin{itemize}

\item We propose a knowledge-based robotic framework that enables robots to selectively request human assistance by jointly reasoning over symbolic knowledge and uncertainty under partial observability.

\item We introduce an uncertainty-aware feedback mechanism that determines both \textit{when} and \textit{what} to ask, allowing robots to obtain only the information necessary to resolve ambiguity during task execution.

\item We develop and integrate a complete robotic system that combines sensing, planning, user interaction, and knowledge management through a unified symbolic knowledge base.

\item We validate the proposed framework through simulation, real-robot experiments, and user studies, demonstrating reduced supervision demands and lower cognitive workload while maintaining task performance.

\end{itemize}
