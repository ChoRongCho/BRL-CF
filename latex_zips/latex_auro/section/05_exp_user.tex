\subsection{User Study}~\label{exp:user_study}
We conducted a Wizard-of-Oz (WoZ) user study to evaluate how different query policies affect task time, answer accuracy, user workload, situation awareness, and cognitive fatigue during human--robot collaboration. Unlike the system evaluation in Section~\ref{exp:sys_eval}, which focused on autonomous task performance, this study focused on the user experience associated with different interaction strategies. Robot behavior was controlled using predefined scenario scripts, while participants interacted with the system through the same tablet-based interface used by the feedback manager.

\subsubsection{Participants}~\label{user:participants}

Twelve participants (7 female and 5 male) took part in the study. Participants ranged in age from 21 to 26 years ($M=23.0$, $SD=1.48$). Each participant received \$20 upon completion of the study.

\subsubsection{Study Design and Conditions}~\label{user:design}

The study employed a within-subject design. Each participant completed ten trials in total, consisting of five waste sorting scenarios and five tomato harvesting scenarios.

In each domain, participants experienced five interaction conditions: \textbf{All}, \textbf{No}, \textbf{Ours1}, \textbf{Ours2}, and \textbf{KnowNo}. \textbf{All} requested feedback whenever a query was available, representing continuous human supervision. \textbf{No} performed the task without requesting user feedback. \textbf{Ours1} used the proposed uncertainty-based state query policy. \textbf{Ours2} used a variant of the proposed policy designed to expose participants to a broader range of query situations. \textbf{KnowNo} served as an action-level query baseline.

Each domain consisted of five scenarios and five interaction conditions. The assignment between scenarios and conditions was randomized for each participant such that every participant experienced all five conditions exactly once per domain. This design prevented a particular condition from being consistently associated with a specific scenario. Detailed scenario descriptions are provided in Appendix~\ref{appx:user}.

\subsubsection{Materials}~\label{user:materials}

Participants interacted with the robot through a tablet-based interface. The interface displayed the current task state, highlighted task-relevant objects, and presented robot-generated questions together with selectable answer options.

The waste sorting scenarios required participants to identify the category of waste objects, including general waste, paper, cans, and plastic. The tomato harvesting scenarios required participants to identify the state of tomatoes, such as whether a tomato was ripe, unripe, or rotten. Example interface screenshots are shown in Fig.~\ref{fig:user_interface}.

To ensure a consistent interaction sequence across participants, we adopted a WoZ protocol. The experimenter advanced the robot state according to predefined scenario scripts and recorded participant responses. If a participant selected an incorrect answer, the response was recorded as incorrect for evaluation purposes, but the scenario continued using the intended task state so that subsequent interactions were not affected by earlier mistakes.

\subsubsection{Procedure}~\label{user:procedure}

Before the experiment, participants received an explanation of the study objectives, the two task domains, and the robot interaction process. They were then introduced to the tablet interface and completed several practice interactions to become familiar with the available information and response procedure.

During the main session, participants completed ten trials without assistance. In each trial, participants observed the robot execution through the interface and responded whenever the system requested feedback. After each scenario, participants completed a questionnaire regarding the interaction experience. After completing all ten trials, participants completed post-study questionnaires and provided qualitative feedback during a short debriefing session.

\subsubsection{Measures}~\label{user:measures}

We collected both subjective and objective measures.

Subjective workload was measured using NASA Raw-TLX. Situation awareness was evaluated using SAGAT-style questionnaires and SART. Cognitive fatigue was measured using a self-reported fatigue questionnaire administered after each task condition. Unless otherwise specified, questionnaire responses were recorded using a 5-point Likert scale.

For SAGAT, each scenario included six multiple-choice questions concerning the current task state, the source of robot uncertainty, and the expected consequences of user inaction. Responses were scored as correct or incorrect using predefined answer keys, and the resulting accuracy was used as the answer accuracy measure and as an objective measure of situation awareness.

In addition, we recorded behavioral measures including task completion time, questionnaire completion time, and participant responses for each condition and scenario. Task completion time was measured as the elapsed operation time for each scenario.

\paragraph{Situation awareness.}
In the aggregated analysis, answer accuracy based on SAGAT scoring did not show a significant overall condition effect, $\chi^2(4)=6.48$, $p=.166$, Kendall's $W=0.14$. The mean accuracy was highest for \textbf{Ours1} ($M=0.67$, $SD=0.10$), followed by \textbf{No} ($M=0.65$, $SD=0.11$), \textbf{All} ($M=0.62$, $SD=0.13$), \textbf{Ours2} ($M=0.61$, $SD=0.12$), and \textbf{KnowNo} ($M=0.59$, $SD=0.17$).

A clearer effect appeared in the waste sorting domain. Answer accuracy differed significantly across conditions, $\chi^2(4)=12.92$, $p=.012$, $W=0.27$. \textbf{Ours1} achieved the highest accuracy ($M=0.72$, $SD=0.15$) and was significantly higher than \textbf{All} ($M=0.56$, $SD=0.19$; Holm-adjusted $p=.027$, rank-biserial $r=1.00$). In tomato harvesting, the condition effect was not significant, $\chi^2(4)=4.06$, $p=.398$, $W=0.08$, with \textbf{Ours2} showing the highest mean accuracy ($M=0.69$, $SD=0.16$).

The item-level SAGAT analysis indicates that the strongest awareness signal came from Q4, which asked why the robot requested information. In waste sorting, Q4 accuracy differed strongly across conditions, $\chi^2(4)=22.12$, $p<.001$, $W=0.46$. \textbf{Ours1} reached perfect accuracy on this item ($M=1.00$, $SD=0.00$), outperforming \textbf{No} ($M=0.17$, $SD=0.39$; Holm-adjusted $p=.012$, $r=1.00$) and \textbf{Ours2} ($M=0.08$, $SD=0.29$; Holm-adjusted $p=.007$, $r=-1.00$ for \textbf{Ours2}--\textbf{Ours1}). The same Q4 pattern remained visible after combining domains, $\chi^2(4)=15.93$, $p=.003$, $W=0.33$, where \textbf{Ours1} exceeded \textbf{No} (Holm-adjusted $p=.014$) and \textbf{Ours2} was lower than \textbf{Ours1} (Holm-adjusted $p=.023$).

\paragraph{Task time.}
Across both domains, \textbf{No} had the shortest operation time ($M=2.81$ min, $SD=0.52$), which reflects the absence of user queries. Among the interactive conditions, \textbf{Ours2} had the shortest average time ($M=3.11$ min, $SD=0.62$), followed by \textbf{Ours1} ($M=3.19$ min, $SD=0.66$), \textbf{KnowNo} ($M=3.33$ min, $SD=0.51$), and \textbf{All} ($M=3.37$ min, $SD=0.53$). The same descriptive pattern appeared in both domains: in waste sorting, \textbf{Ours2} averaged $2.55$ min and \textbf{Ours1} averaged $2.62$ min, compared with $2.91$ min for \textbf{KnowNo} and $2.92$ min for \textbf{All}; in tomato harvesting, \textbf{Ours2} averaged $3.67$ min and \textbf{Ours1} averaged $3.75$ min, compared with $3.74$ min for \textbf{KnowNo} and $3.81$ min for \textbf{All}.

\paragraph{Workload, effort, and fatigue.}
NASA-TLX workload showed a significant condition effect in the aggregated data, $\chi^2(4)=18.87$, $p<.001$, $W=0.39$. \textbf{Ours2} produced lower workload ($M=1.85$, $SD=0.64$) than \textbf{Ours1} ($M=2.07$, $SD=0.79$) and \textbf{KnowNo} ($M=2.15$, $SD=0.73$). The \textbf{Ours2}--\textbf{KnowNo} comparison was significant after Holm correction ($p=.007$, $r=-1.00$), while the \textbf{Ours2}--\textbf{Ours1} comparison showed the same direction but did not survive correction ($p=.059$, $r=-0.93$). \textbf{Ours1} was higher than \textbf{No} ($p=.050$, $r=0.85$), which is expected because \textbf{No} involved no feedback requests and therefore less interaction.

Effort followed the same pattern. The aggregated NASA effort item showed a significant condition effect, $\chi^2(4)=21.86$, $p<.001$, $W=0.46$. \textbf{Ours2} required less effort than \textbf{All} ($p=.047$, $r=-1.00$) and \textbf{KnowNo} ($p=.007$, $r=-1.00$). In waste sorting, \textbf{Ours2} also reduced effort relative to \textbf{KnowNo} ($M=2.58$ vs. $M=4.00$; Holm-adjusted $p=.014$, $r=-1.00$). Fatigue did not show a significant condition effect in the aggregated analysis, $\chi^2(4)=8.16$, $p=.086$, $W=0.17$, but the means were consistent with the workload pattern: \textbf{Ours2} ($M=2.67$) was lower than \textbf{Ours1} ($M=3.04$) and \textbf{KnowNo} ($M=3.08$).

\paragraph{SART.}
The SART measures did not show significant condition effects in the aggregated analysis. Demand was comparable across conditions, $\chi^2(4)=3.89$, $p=.421$, $W=0.08$, while supply, $\chi^2(4)=0.30$, $p=.990$, $W=0.01$, and understanding, $\chi^2(4)=1.79$, $p=.775$, $W=0.04$, remained high across all conditions. These results suggest that the proposed query policies did not reduce participants' perceived situation-awareness resources or understanding, even when the amount and timing of user interaction changed.

\subsubsection{Discussion}~\label{user:discussion}

The user-study results show a tradeoff between awareness and workload. \textbf{Ours1} produced the strongest SAGAT performance in waste sorting, especially for questions about the reason for robot uncertainty. This suggests that state-level uncertainty queries can make the robot's information needs more legible to users: participants were not only answering questions, but also understanding why the robot asked them.

At the same time, \textbf{Ours2} reduced subjective workload and effort relative to the action-level \textbf{KnowNo} baseline, and reduced effort relative to continuous supervision. This pattern suggests that selectively presenting state-relevant questions can lower interaction cost when compared with asking users to judge action-level decisions. The low workload of \textbf{No} should be interpreted carefully, because it reflects the absence of interaction rather than a desirable collaborative strategy; in the system evaluation, no-feedback execution substantially reduced task success in the more uncertain tomato harvesting task.

Overall, the user study supports the system-level results in Section~\ref{exp:sys_eval}. Selective querying can preserve or improve users' understanding of robot uncertainty while avoiding the workload associated with continuous or action-level feedback. The difference between \textbf{Ours1} and \textbf{Ours2} also indicates that query timing and query content should be tuned jointly: queries that clearly expose the source of uncertainty can improve awareness, while reducing redundant or difficult interactions can lower workload.
